Grad Platform — API Documentation

Full reference for all REST endpoints including request/response shapes and test cases.

Base URL: All routes are relative to your server root, e.g. https://api.example.com/api/…
Authentication: Protected endpoints require an Authorization: Bearer <JWT> header.
Content-Type: All request bodies must be application/json.
AUTH Shared Auth — /api/auth
GET /api/auth/me Get the current logged-in user's profile
🔒 JWT Required
Response 200
{
  "id": "guid",
  "firstname": "string",
  "lastname": "string",
  "email": "string",
  "language_pref": "string",
  "device_id": "string",
  "student": { "parent_email": "string", "academicLevel": "string", "academicYear": 0 } | null,
  "teacher": { "subject": "string", "is_approved": true } | null
}
Test Cases
✔ TC-01 — Valid JWT (Student)
Send request with a valid student JWT → 200 with student populated, teacher null.
✔ TC-02 — Valid JWT (Teacher)
Send with a valid teacher JWT → 200 with teacher populated, student null.
✘ TC-03 — No token
Omit Authorization header → 401 Unauthorized.
PUT /api/auth/update Update profile (shared for all roles)
🔒 JWT Required
Request Body
FieldTypeNotes FirstNamestring?Optional LastNamestring?Optional LanguagePrefstring?e.g. "en", "ar" DeviceIdstring?Push token Passwordstring?New password (optional) AcademicLevelstring?Students only AcademicYearint?Students only ParentEmailstring?Students only Subjectstring?Teachers only
Responses
200 — { "message": "Profile updated successfully" }
401 — Missing/invalid JWT
Test Cases
✔ TC-01 — Update first name only
Body: {"FirstName":"Ahmed"} → 200, only first name changed.
✔ TC-02 — Change password
Body: {"Password":"NewPass@123"} → 200, subsequent login uses new password.
✘ TC-03 — No token
→ 401 Unauthorized.
POST /api/auth/forgot-password Request password reset token
Request Body
FieldTypeNotes Emailstringrequired
Responses
200 — { "message": "Password reset link sent if email exists." } (always 200 to prevent enumeration)
400 — Email field is empty
Test Cases
✔ TC-01 — Existing email
→ 200. Token generated (logged to console; email sending pending).
✔ TC-02 — Non-existent email
→ 200 (same message — no user enumeration).
✘ TC-03 — Empty body
→ 400 "Email is required."
POST /api/auth/reset-password Apply a password reset token
Request Body
FieldTypeNotes Emailstringrequired Tokenstringrequired from forgot-password NewPasswordstringrequired
Responses
200 — { "message": "Password has been reset successfully." }
400 — Invalid token or missing fields
Test Cases
✔ TC-01 — Valid token & new password
→ 200. Can now log in with new password.
✘ TC-02 — Expired/wrong token
→ 400 with error descriptions from Identity.
